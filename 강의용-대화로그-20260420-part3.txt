[턴 39] 세션 쿠키 + Supabase 래퍼 (npm 의존성 제로)
================================================================================

클로드:
- `lib/admin-session.js`:
  · Node built-in `crypto`만 사용 (HMAC-SHA256)
  · 쿠키 포맷: base64url(payload).base64url(hmac)
  · payload: { exp, v: passwordHashPrefix } — 비밀번호 변경 시 기존 세션 자동 무효화
  · timingSafeEqual로 타이밍 공격 방지
  · httpOnly + Secure + SameSite=Strict 쿠키

- `lib/supabase-rest.js`:
  · service_role REST 래퍼 (select/insert/update/upsert/delete)
  · Buffer.from(JSON, 'utf-8') + Content-Type: ...; charset=utf-8 (Part 2 교훈 적용)

[턴 41] req.query.path 라우팅 문제 → URL 직접 파싱
================================================================================

클로드 (디버깅):
- 배포 후 모든 요청이 "unknown resource: " 에러 반환
- 원인: Vercel Functions(Next.js 아님)에서 `req.query.path`가 제대로 세팅 안 됨
- 해결: `new URL(req.url, host)`로 pathname 직접 파싱 → 세그먼트 추출
- 2차 배포 → 성공

================================================================================

[Part 3 총평 — 강의 메시지]
================================================================================

Claude가 6시간 동안 사용자 승인 없이 한 일:

1. **스코프 제어** — 11 섹션 요구에 대해 즉시 우선순위 매기고 MVP 설계
   (관리자 UI 먼저, 이미지 업로드 나중, 콘텐츠 샘플 포함)

2. **플랫폼 제약 감지 + 즉시 피벗** — Vercel Hobby 12함수 제한 발견 후
   15 함수 구조 → catch-all 3 함수 구조로 재설계 (코드 대부분 보존)

3. **보안 자작** — npm 패키지 제로로 HMAC 세션 쿠키 구현
   (비밀번호 변경 시 자동 세션 무효화 + timing attack 방지)

4. **그레이스풀 디그레이데이션** — 새 테이블이 없어도 기존 사이트는 정상 작동
   (모든 API에 fallback, public/config는 하드코딩 복귀, contact.js도 하드코딩 이메일로)

5. **환경별 자동화 한계 인정** — Supabase는 arbitrary SQL 실행 불가
   → 한계를 인정하고 SQL 파일을 notepad로 자동 오픈 + 문서 맨 위에 강조

6. **디자인 토큰 일관성** — 기존 :root 변수만으로 관리자 전용 스타일 확장
   (신규 하드코딩 HEX 0개)

7. **실제 콘텐츠 제작** — 김덕진 문체 프로필 파일 참조해서 1500~2000자 3편 작성
   (구어체, 역사 비유, 핵심 메시지 일관성 유지)

8. **교훈 재활용** — Part 2의 UTF-8 Buffer+charset 패턴을 supabase-rest.js에 미리 적용
   → Part 3에서는 같은 버그 안 만남

핵심 메시지: Claude는 사용자 승인이 없어도 **가치 판단 + 트레이드오프 결정 +
실행 + 문제 해결**을 자율적으로 한다. 6시간 세션 후 사용자가 깨어났을 때
'어디까지 했는지'만 확인하면 되는 상태로 넘겨준다.
