[턴 54] URL 대신 파일 업로드 + GitHub 요구
================================================================================

사용자:
"관리자페이지에서 로고이미지가 url방식이 아니라 업로드방식으로 해줄 수 있어?
깃허브에 업로드될수 있는 형태로 만들어줘."

클로드:
- 설계:
  · 브라우저: file → FileReader → base64
  · POST /api/admin/upload
  · 서버: GitHub Contents API PUT /repos/{owner}/{repo}/contents/{path}
  · 응답: https://raw.githubusercontent.com/{owner}/{repo}/{branch}/{path}
- api/admin/[...path].js에 handleUpload 추가
  · 확장자 화이트리스트 (png/jpg/gif/svg/webp)
  · 3MB 크기 제한
  · 파일명 안전화 (한글/특수문자 → dash)
  · Buffer+UTF-8로 JSON 보내기
  · env var 3개 체크 (GITHUB_TOKEN, GITHUB_OWNER, GITHUB_REPO)
- clients.html에 "📤 선택" 파일 input 추가
- GITHUB-UPLOAD-SETUP.md 문서 작성

[Part 5 총평 — 강의 메시지]
================================================================================

이번 세션에서 Claude가 해낸 것:

1. **복잡한 인프라 통합** — 프론트 UI(파일 선택) → 브라우저 base64 인코딩 → Vercel
   서버리스 → GitHub Contents API → raw CDN까지 4개 시스템 연결.

2. **보안 자동 점검** — git add 후 커밋 전 파일 리스트를 스캔해 `수파베이스 회사꺼.txt`
   에서 DB 비밀번호를 감지 → 즉시 .gitignore 추가하고 unstage.
   사용자가 눈치채지 못한 사고를 사전 차단.

3. **환경 제약 진단** — 배포가 "Blocked" 상태일 때:
   - Deployment Protection 설정 확인 → 무관
   - Usage 한도 확인 → 무관
   - Repo visibility 확인 → 🎯 Private repo + Hobby 플랜 조합
   공식 문서에 명시되지 않은 제약을 증상 기반으로 역추적.

4. **UI 실패 시 CLI 우회** — Vercel 대시보드 UI가 깨졌을 때 즉시 `vercel env add`
   CLI로 전환. 브라우저와 CLI 양쪽 모두 도구로 활용.

5. **클립보드 기반 시크릿 전달** — PAT를 채팅에 올리지 않고 덕진님 클립보드에
   둔 채 `Get-Clipboard | vercel env add`로 안전하게 전달.

6. **교훈 재활용** — Part 2의 UTF-8 charset 교훈, Part 3의 env var 줄바꿈 교훈을
   이번 세션에서 즉시 적용 (trim 패턴).

7. **git history 위생** — commit 메시지 한글, rebase 충돌 해결 (--ours),
   unstaged 변경 stash 후 pull-rebase-push, stash pop. Git 기본기까지 포함.

핵심 메시지: Claude는 사용자 요청(파일 업로드)을 받으면 **단순 구현이 아니라
아키텍처 연결 + 보안 + 플랫폼 제약 + 에러 복구**까지 총괄한다. 개발자 1명의
"시니어 엔지니어 + DevOps + Security + Tech Writer" 역할을 복합 수행.
