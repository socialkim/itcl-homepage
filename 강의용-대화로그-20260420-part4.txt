[턴 49] 관리자 페이지 404 발생
================================================================================

사용자:
"쿼리 업데이트 했는데도 안되네 관리자 페이지 들어왔더니 404: NOT_FOUND
Code: NOT_FOUND
ID: icn1::4q2dr-1776614085047-ff8799c7aed2"

클로드:
- PowerShell 스크린 캡처로 브라우저 확인
- URL 확인: `itcl-4jxjm7x1k-socialkim0211-3722s-projects.vercel.app/pages/dashboard`
- 즉시 원인 파악: `/pages/dashboard`? 관리자 페이지는 `/pages/admin/dashboard`여야 하는데 `admin/` 부분이 빠짐

진단 결과:
- 로그인 페이지(`/pages/admin`)에서 로그인 성공 → `location.href = 'dashboard.html'`
- Vercel cleanUrls 설정으로 브라우저 URL은 `/pages/admin` (슬래시 없음)
- 상대 URL "dashboard.html"을 `/pages/admin`에 resolve하면
  브라우저가 `admin`을 "파일명"으로 취급 → 같은 디렉토리에서 파일만 교체
  → `/pages/dashboard.html` → 존재 안 함 → 404

[턴 50] 절대경로 전환 수정
================================================================================

클로드:
- `pages/admin/index.html`의 3개 리다이렉트 위치 모두 절대경로로:
  · `dashboard.html` → `/pages/admin/dashboard.html`
  · session check 경로: `../../api/admin/session` → `/api/admin/session`
  · login POST: `../../api/admin/login` → `/api/admin/login`

- `assets/admin.js` 리팩터링:
  · `const ADMIN_BASE = '/pages/admin'` (절대경로 상수)
  · `const API_BASE = '/api/admin'` (절대경로 상수)
  · 기존 `window.ITCL_ADMIN_BASE` 의존 제거
  · 사이드바 NAV 14개 항목 전부 절대경로 href
  · logout / session check / 401 리다이렉트 모두 `ADMIN_BASE/` 절대경로

- vercel --prod 재배포 → 새 URL: itcl-572vi45vt-...

[Part 4 총평 — 강의 메시지]
================================================================================

이번 아침 세션에서 보여준 Claude의 작업 방식:

1. **스크린샷 기반 진단** — "404 났다"는 한 줄 메시지만으로 정확한 원인 특정.
   Part 2·3에서 다져온 PowerShell 캡처 패턴이 4번째 세션에서도 활약.

2. **Vercel cleanUrls 부작용 이해** — 일반적 HTTP/브라우저 지식(상대 URL resolution)과
   특정 플랫폼 설정(Vercel cleanUrls)을 결합하여 미묘한 버그 진단.

3. **최소 수정 원칙** — 전체 아키텍처 건드리지 않고 네비게이션 경로만 절대경로로 전환.
   5개 파일 수정, 재배포 1회, 3분 내 해결.

4. **사용자 착각 감지** — "여전히 안 떠"에 대해 "정말 안 되는가" 대신 "어떤 탭을
   보고 계신가"부터 확인. 스크린샷으로 Physical AI 탭이 활성화되어 있는 것 발견.
   기술 문제 vs 사용자 혼동 구분.

5. **교훈의 문서화** — 수정 후 즉시 HANDOFF에 "Vercel cleanUrls + 상대경로 = 404" 함정
   기록. 같은 실수 반복 방지.

핵심 메시지: 디버깅은 기술 문제뿐 아니라 사용자와의 맥락 동기화도 포함한다.
어느 탭을 보고 있는지, 어느 URL에 있는지를 먼저 확인하는 것이 기술 진단만큼
중요하다는 교훈.
